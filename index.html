What It Does

This is a browser-based imitation, invention, counterpoint, and Fishtail Fugue MIDI generator. It creates a structured piece from sections of bars, keys, modes, metres, cadences, voice-leading rules, breath/rest choices, and weighted randomness.

The generator aims for calm, coherent, harmonically pleasing material rather than completely wild note soup. It writes a MIDI file, a project JSON recipe, and generation notes with musical observations, form details, and kind little Gemma reassurances.

Quick Start

  1. Choose sections in Form: bars, key, mode, meter, cadence, role, and treatment.
  2. Choose Imitation + Invention, Counterpoint, or Fishtail Fugue, then set voice count, pedal voices, breathing, density, and pitch mode.
  3. Press Generate + Save MIDI. The MIDI file should download automatically.
  4. Use Save Project JSON to keep the exact recipe and seed for later.

Probe And Time

The Probe + Time panel is the live sound and conductor-timing area. The page loads silent. Turning on Probe sound starts the Teardrop reference tone at the current Reference Hz. Probe pitch chooses the reference note, and Fine pitch trims cents by ear; both write into the same Reference Hz used by MIDI generation.

Living Reference listens only after you press Listen for pitch. Play or sing one sustained monophonic note, then press Use this pitch to freeze the recent pitch centre as Fishtail's exact reference. The input is analysed locally through a silent graph; it is not monitored, recorded, uploaded, or saved with device identity.

Metronome starts a louder pink-noise ticker using the preview meter, meter accents, and the rational swing / irrational swing controls. When Preview meter says Follow section 1, it takes the time signature from the first Form row. Swing starts straight by default; DUB mode sets a gentle strolling swing. Tempo lattice writes the same pulse timing into the MIDI conductor track as Set Tempo events, while the notes themselves stay on their formal grid.

Plain Equal Temperament MIDI exports ordinary note numbers, so exact captured acoustic tuning depends on the receiving synth. Amy Dub Intonation, Bend MIDI, project JSON, Probe WAV, and Fishtail's tempo/reference metadata preserve the captured relationship more directly.

Prepare probe WAV and Prepare ticker WAV render optional mono 24-bit stems after MIDI generation. Ticker WAV export is peak-normalized to -6 dBFS for DAW-friendly headroom. Stems are saved only when their explicit Save WAV buttons are pressed.

Randomness

The app runs locally in the browser. It uses the browser Web Crypto random source to make a seed, then uses that seed for repeatable musical choices inside the generator. On an ordinary web page, JavaScript cannot read /dev/urandom directly, but modern browsers expose secure random bytes through crypto.getRandomValues. Advanced users can optionally mix in a custom entropy endpoint; see PUBLISHING.md.

D4 rolls a gentler form with a tasteful chance of Refrain or Gentle Development after the first section. D20 rolls a wider form with more varied keys, meters, Refrain returns, Development rows, and occasional Dubby treatments. The first section stays Fishtail/Source so later rows have source material to return to or develop.

Pitch Modes

Equal Temperament writes ordinary MIDI notes. This is the safest mode for most synths and DAWs. The voice tracks stay clean: note on/off events only, no program changes, no controller setup, and no track-name clutter. If Tempo map is on, a separate conductor track writes BPM and time signatures. The selected BPM is also shown in the downloaded filename.

Amy Dub Intonation is Amy McBride's experimental ratio-based dub tuning. Instead of dividing the octave into twelve equal steps, it maps chromatic slots to chosen ratios. It is designed to be run through Entonal or a similar retuner that can define these ratios. In retuner mode the MIDI notes are carrier keys: the note number may look ordinary or unusual, but the retuner turns that key into the intended ratio pitch.

Amy Dub ratio pattern, chromatic from the selected root:

  • 0 1/1 home
  • 1 16/15 upper bite
  • 2 9/8 motion
  • 3 8/7 blue low
  • 4 7/6 minor soul
  • 5 5/4 sweet light
  • 6 4/3 plagal
  • 7 11/8 shimmer
  • 8 3/2 pillar
  • 9 8/5 pivot
  • 10 13/8 tender lift
  • 11 7/4 dub crown

Bend MIDI writes multiple single-voice MIDI tracks, one voice per track/channel, and retunes each voice toward Amy Dub intonation with pitch-bend messages. It should be played through separate mono instruments or separate mono channels. If the parts are merged into one polyphonic instrument, the pitch bends will fight each other and the tuning will not work properly.

Fishtail Gravity

Fishtail Gravity is Amy's compositional idea for making complex theory flow gently. Notes are treated as forces with tendencies: leading tones want to rise, certain minor-key colours want to fall, cadences pull the voices home, bass tones create weight, and silence lets another voice answer.

The generator combines that gravity with counterpoint, fugue structure, rhythmic breathing, and optional Amy Dub ratios. Equal Temperament keeps the usual twelve-step tuning; Amy Dub Intonation keeps the same musical functions but lets the pitch world bend toward Amy's ratio experiment.

Modes And Gravity

Major, harmonic minor, modal scales, and gravity melodic minor are available. Gravity melodic minor allows extra minor-key passing tones with tendency rules: leading tones rise, raised sixths lift, flat sevenths fall, and flat sixths resolve down.

Fishtail Fugue uses the Form rows as tonal regions, then shapes them into exposition, episodes, middle entries, final return, and cadence. If the form is too short, the generator quietly adds enough internal sections or bars and explains the shaping in Generation Notes. This means a fugue can come out longer than the visible Form. With DUB off it uses Formal Gravity; with DUB on it keeps the fugue map but lets the bass and offbeat answers breathe more freely.

Dub Gravity keeps the counterpoint engine but biases the bass toward repeated root/fifth gravity, gives middle voices offbeat skank gestures, makes the upper voices breathe more, and allows the occasional parallel-perfect bend in the rules while noting it gently in Generation Notes. The interface also shifts into a black and green terminal-style view while DUB is armed.

Refrains, Pedals, And Held Notes

Role chooses what each Form row does. The first row is always Fishtail, because it introduces the source material. Later Refrain rows return that first-section idea, and later Development rows reshape it into a new answer. If a Development ever has no source to work from, it writes a fresh Fishtail section and says so in Generation Notes.

Treatment chooses the flavour of a Refrain or Development. Source marks the first Fishtail row. Clear Return keeps a Refrain close to the first-section source. Gentle develops it with key movement, octave changes, small answering gestures, light rhythm changes, and voice handoff. Dubby keeps recognizable fragments while the bass finds more root/fifth gravity and the inner voices answer with offbeat skank chords.

Pedal voice switches tell the generator that a voice is allowed to hold longer tonic or fifth notes as an intentional drone. Bass pedal turns on by default in DUB mode. For melody and inner voices, long held notes are shaped to breathe and resolve musically, so they feel like expressive tension rather than accidental stuck notes.

Project JSON remembers section roles, treatments, pedal choices, Refrain source material, and Development transformations, so a favourite form can be saved and returned to later.

Conceptual Inspirations

  • Fux-style species counterpoint and Palestrina-style voice-leading practice.
  • Knud Jeppesen-style Palestrina counterpoint as a stylistic reference.
  • Backtracking counterpoint generators that place notes, validate rules, and retry.
  • Constraint-programming approaches such as FuxCP, used here only as a design reference for explicit hard rules.
  • Markov/probabilistic Palestrina approaches such as Farbood and Schoner, used here only as inspiration for combining deterministic rules with weighted musical choices.
  • Interactive rule-checking tools such as CounterPointer, used here only as inspiration for producing a human-readable report.

Implementation Notes

  • This version was vibe coded with Codex, collaborating as Lambda Echo.
  • The MIDI writer in src/app.js is original code.
  • The Teardrop Probe, Living Reference Input, pink-noise ticker, and tempo-lattice conductor are implemented as browser-native Web Audio and Standard MIDI logic, translated from Amy's SuperCollider reference sketches in this repository.
  • The moving torus visualisation uses Three.js, an MIT-licensed open-source 3D library.
  • The seeded random functions are small public-domain-style PRNG/hash patterns commonly circulated in JavaScript communities; they are included inline only for deterministic replay after the browser supplies a system entropy seed.
  • Browser APIs used: Canvas, WebGL through Three.js, optional Device Motion/Orientation input, Web Audio, optional MediaDevices audio input, optional Vibration API feedback, Blob download, TextEncoder, and Web Crypto crypto.getRandomValues.
